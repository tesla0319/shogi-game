# test-design スキル

## 目的

変更対象の関数・機能に対して、正常系・異常系・境界値・回帰・非機能の観点でテストケースを洗い出し、
pytestで実装可能な粒度のテスト観点表として返す。
デフォルトの成果物は観点表まで。pytestコードの実装は依頼されたときのみ行う。

## 原則

- CLAUDE.md の共通ルール（開発方針・コード方針・テスト方針・やらないこと）に従う
- 1ケース = 1観点（pytestの1テスト関数に対応する粒度）で洗い出す
- 各ケースに優先度を付ける:
  - **Must**: 無いとバグを見逃すケース
  - **Should**: 境界の安心材料になるケース
  - **Could**: 非機能など、無くてもMVPとしては困らないケース
- 仕様が曖昧な項目は推測でテスト設計せず、「前提・未確定事項」として明示する

## 手順

### 1. 対象の確認
- テスト対象（関数・機能・Phase全体）と、その仕様（入力・出力・失敗時の挙動）を確認する
- 仕様が曖昧な項目は「テスト不能」として前提・未確定事項に回す

### 2. 入力空間の整理
- 引数・外部入力・状態（ファイル有無・設定値）を列挙し、それぞれの有効値・無効値・境界を出す

### 3. 観点別に洗い出し
- 下記チェックリスト順にケースを列挙する

### 4. 回帰の特定
- 今回の変更で挙動が変わりうる既存機能を挙げ、既存テストでカバーされているか確認する

### 5. 優先度付け
- 各ケースを Must/Should/Could に分類する

### 6. 報告
- 観点表で出力する。依頼があればpytestスケルトン（test関数名 + docstring + assert方針のみ）も出す

## チェックリスト

- [ ] **正常系**: 代表的な入力で期待出力が得られるか（最低1ケース、入力パターンが複数あれば各1）
- [ ] **異常系**: 以下の失敗時に、CLAUDE.mdの型どおりの挙動（例: Noneで返す）になるか
  - 不正入力（型違い・不正な値）
  - ファイル無し・権限なし
  - 外部プロセス失敗・タイムアウト
  - CLI入力不正（存在しないサブコマンド・未定義コマンド）
  - コマンドライン引数不正（必須引数の欠落・型違い・不正な組み合わせ）
  - 対話入力の異常値（空Enter・中断（Ctrl+C/EOF）・想定外の文字列）
- [ ] **境界値**: 空文字/空リスト・0・1・最大値・最大値+1・重複・前後空白・型違い
- [ ] **回帰**: 変更に隣接する既存機能が壊れていないか（既存テストで足りるなら「既存でカバー」と明示）
- [ ] **状態依存**: 実行順序・ファイルの有無・設定値の違いで結果が変わるケース
- [ ] **非機能（MVPの範囲で）**: 明らかな性能劣化（ループ内I/O等）・秘密情報のログ出力、の2点のみ確認。負荷試験等はしない
- [ ] **テスト容易性**: テストしにくい構造（時刻直接参照・グローバル状態）があれば、テスト側の工夫または最小の設計変更提案を1つだけ出す
- [ ] **pytest粒度**: 1テスト1アサーション群、parametrize可能な同型ケースはまとめて提案、テスト名は `test_<対象>_<条件>_<期待>` 形式

## 出力フォーマット

```markdown
## テスト観点表: <対象>

### 前提・未確定事項
<仕様が曖昧でテスト設計できなかった点。無ければ「なし」>

### 観点表
| # | 分類 | 優先度 | ケース名(test_...) | 入力・条件 | 期待結果 |
|---|---|---|---|---|---|

### 回帰確認
<影響を受けうる既存機能と、既存テストでのカバー状況>

### 実装メモ
<parametrize推奨のまとまり、fixture化候補、モック方針など pytest実装時の注意>
```

## 適用例

- 「save_config関数のテスト観点を出して」
- phase-implementationの手順3（テスト）で観点を出すとき
- quality-reviewが「テスト充足: 不足」と指摘した後の詳細設計
- 「このPhaseのテスト、漏れがないか確認して」
- CLIツールの引数まわり・対話入力まわりのテスト観点出し

## 適用しないケース

- テスト実行やCI設定の作業そのもの
- E2E・UI自動化・負荷試験の詳細設計（MVP範囲外。必要なら別スキル化を提案）
- 仕様が未確定のままのテスト設計（先に仕様確認へ戻す）
- 既存テスト全体の網羅率監査（大規模作業の入口になるため。対象は今回の変更範囲に限定）
